Verdex Chain

A high-performance Layer 1 blockchain purpose-built for decentralized trading. Native VDX token. PRC20 token standard. Verdex DEX at the core.

Phases 0–3 live. Verdex Testnet (chain ID 7201), faucet, explorer, VP mining, and PRC20 Solidity contracts are available. Phase 4 (DEX swap) is not deployed yet — Router/Factory sections below are the planned interface.

⚠️ Current Architecture: The hosted RPC (/api/rpc) is backed by Supabase and supports read-only eth_* methods (chain ID, balances, blocks). It does not execute EVM bytecode or process eth_sendRawTransaction yet. A full Geth Clique PoA testnet is in development. Hardhat deploy will work once the Geth node is live. PRC20 contracts compile against Hardhat's local network today.

What is VerdexSwap and how does the ecosystem work?

VerdexSwap is a decentralized finance (DeFi) ecosystem built on a custom L1 blockchain. It consists of a DePIN points mining network, a testnet L1 with native VDX gas, and the PRC20 token standard for developers. The automated market maker (AMM) including Swap, Pool, and Farm ships in Phase 4.

The PRC20 token standard is identical to ERC-20 at the ABI level. Use standard OpenZeppelin ERC20 templates and Hardhat. The hosted JSON-RPC is available at https://verdex-ecosystem-production.up.railway.app/rpc (read-only currently). A full EVM Geth node is currently in development.

Key Properties

  • Native token: VDX (gas, governance, staking, future base pair)
  • Token standard: PRC20 (ERC-20 compatible Solidity)
  • Consensus (testnet): Proof of Authority — single validator (PoS roadmap deferred)
  • Tooling: MetaMask (add network), Hardhat (local compile + test), JSON-RPC eth_* (read-only until Geth live)
  • Chain ID: 7201 (0x1c21)
  • Block time: ~5s hosted ledger / ~2–5s on upcoming Geth Clique testnet

What is the PRC20 Token Standard?

PRC20 is the fungible token standard natively built for the Verdex Chain. It is fully backwards-compatible with the Ethereum ERC20 interface, meaning existing Solidity tooling, wallets, and libraries like OpenZeppelin work natively out of the box.

What does the PRC20 Solidity interface look like?

Below is the complete PRC20 interface. Since it maps 1:1 with ERC20, all core functions like transfer, approve, and transferFrom are identical. All functions are required unless marked optional.

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

interface IPRC20 {

    // Returns the total token supply
    function totalSupply() external view returns (uint256);

    // Returns the balance of an account
    function balanceOf(address account) external view returns (uint256);

    // Transfers tokens to a recipient
    function transfer(address to, uint256 amount) external returns (bool);

    // Returns the remaining allowance for a spender
    function allowance(address owner, address spender) external view returns (uint256);

    // Approves a spender to transfer tokens
    function approve(address spender, uint256 amount) external returns (bool);

    // Transfers tokens from sender to recipient using allowance
    function transferFrom(address from, address to, uint256 amount) external returns (bool);

    // Events
    event Transfer(address indexed from, address indexed to, uint256 value);
    event Approval(address indexed owner, address indexed spender, uint256 value);
}

// Optional metadata extension
interface IPRC20Metadata is IPRC20 {
    function name() external view returns (string memory);
    function symbol() external view returns (string memory);
    function decimals() external view returns (uint8);
}

// Optional burn extension
interface IPRC20Burnable is IPRC20 {
    function burn(uint256 amount) external;
    function burnFrom(address account, uint256 amount) external;
}

// Optional mint extension (restricted to owner/minter role)
interface IPRC20Mintable is IPRC20 {
    function mint(address to, uint256 amount) external;
}

How do I implement a PRC20 token in Solidity?

This is a complete, production-ready PRC20 token contract written in Solidity. It can be compiled using solc 0.8.20 and deployed directly to the Verdex testnet via Hardhat.

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

contract PRC20Token {
    string public name;
    string public symbol;
    uint8 public decimals = 18;
    uint256 public totalSupply;

    mapping(address => uint256) public balanceOf;
    mapping(address => mapping(address => uint256)) public allowance;

    event Transfer(address indexed from, address indexed to, uint256 value);
    event Approval(address indexed owner, address indexed spender, uint256 value);

    constructor(string memory _name, string memory _symbol, uint256 _totalSupply) {
        name = _name;
        symbol = _symbol;
        totalSupply = _totalSupply * 10 ** decimals;
        balanceOf[msg.sender] = totalSupply;
        emit Transfer(address(0), msg.sender, totalSupply);
    }

    function transfer(address to, uint256 amount) external returns (bool) {
        require(balanceOf[msg.sender] >= amount, "Insufficient balance");
        balanceOf[msg.sender] -= amount;
        balanceOf[to] += amount;
        emit Transfer(msg.sender, to, amount);
        return true;
    }

    function approve(address spender, uint256 amount) external returns (bool) {
        allowance[msg.sender][spender] = amount;
        emit Approval(msg.sender, spender, amount);
        return true;
    }

    function transferFrom(address from, address to, uint256 amount) external returns (bool) {
        require(balanceOf[from] >= amount, "Insufficient balance");
        require(allowance[from][msg.sender] >= amount, "Insufficient allowance");
        allowance[from][msg.sender] -= amount;
        balanceOf[from] -= amount;
        balanceOf[to] += amount;
        emit Transfer(from, to, amount);
        return true;
    }
}

What are the steps to deploy a PRC20 token on the Verdex network?

Follow this step-by-step developer guide to write, compile, and deploy a custom PRC20 token to the live Verdex blockchain using Hardhat.

Prerequisites

  • Node.js 18+
  • Hardhat or Foundry installed
  • Testnet VDX from the faucet
  • MetaMask connected to Verdex Testnet

Steps

# 1. Initialize project
mkdir my-prc20-token && cd my-prc20-token
npx hardhat init

# 2. Install OpenZeppelin (PRC20 is ERC20-compatible)
npm install @openzeppelin/contracts

# 3. Create your token contract at contracts/MyToken.sol
# 4. Configure hardhat.config.js with Verdex network
# 5. Write deploy script

npx hardhat run scripts/deploy.js --network Verdex

# 6. Verify on Verdex Explorer
npx hardhat verify --network Verdex DEPLOYED_ADDRESS "TokenName" "TKN" "1000000"

Deploy Script

// scripts/deploy.js
const { ethers } = require("hardhat");

async function main() {
  const Token = await ethers.getContractFactory("PRC20Token");
  const token = await Token.deploy("My Token", "MTK", 1000000);
  await token.waitForDeployment();

  console.log(`Token deployed to: ${token.target}`);
}

main().catch((error) => {
  console.error(error);
  process.exitCode = 1;
});

LP Tokens

Phase 4 — not deployed When you add liquidity to a Verdex pool, you will receive LP (Liquidity Provider) tokens. These are PRC20-compatible tokens representing your share of the pool.

  • Burning LP tokens redeems your proportional share of both assets
  • LP tokens can be staked in Verdex Farms for additional VDX rewards

What are the best practices for PRC20 developers?

  • Always use SafeMath or Solidity 0.8+ built-in overflow checks
  • Implement reentrancy guards on state-changing functions
  • Use approve + transferFrom pattern for DEX integrations
  • Set reasonable initial supply (avoid ultra-high supplies that confuse price display)
  • Emit Transfer event from address(0) on mint for proper explorer indexing
  • Consider adding a burn function for deflationary mechanics
  • Test on Verdex Testnet before any mainnet deployment
  • Get an independent audit for any token handling user funds
